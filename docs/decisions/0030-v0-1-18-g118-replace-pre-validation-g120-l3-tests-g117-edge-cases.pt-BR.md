# ADR-0030: v0.1.18 — replace pre-validates root paths, G120 L3 test, G117 edge cases

- **Status**: Accepted
- **Date**: 2026-06-14
- **Context**: Three follow-ups to gaps already closed in v0.1.15/v0.1.16/v0.1.17, surfaced during the user's gap audit. (1) `cmd_replace` resolved the workspace but built the `ignore::WalkBuilder` from `args.paths[0]` WITHOUT pre-validating the root against the jail. A relative `../escape` resolved against CWD (not workspace), and per-entry validation in the worker thread only emitted one `JailViolation` event per file walked — the user-facing diagnostic was buried under N events and the walker wasted I/O. (2) G120 L1/L2/L4 were covered by integration tests but L3 (the cross-validation `--append + --expect-checksum + empty stdin + --allow-empty-stdin` path) had no end-to-end test. (3) G117 fuzzy cascade had 9 strategies and multi-pair covered, but edge cases — Unicode (UTF-8 diacritics), CRLF line-ending preservation, multi-pair where the same `--old` appears multiple times — were not explicitly covered.
- **Decision**:
  1. **G118 fix for replace** — in `src/commands/replace.rs::cmd_replace`, after `global.resolve_workspace()`, iterate over `args.paths` and call `crate::path_safety::validate_path(path, &workspace)?` for each BEFORE constructing the `WalkBuilder`. Fail fast with `WORKSPACE_JAIL` (exit 126) on the first violation. This mirrors the resolve-first convention adopted in v0.1.15 for `write` (ADR-0027) and propagated to `edit` / `copy` / `apply` / `move` / `rollback` / `set` / `del` / `case` — `replace` was the last mutating command still relying on per-entry validation.
  2. **G120 L3 integration test** — `g120_l3_append_empty_stdin_with_matching_checksum_succeeds`: writes a seeded file, hashes it, runs `write --append --allow-empty-stdin --expect-checksum <HASH> < /dev/null`, asserts exit 0, asserts `stdin_bytes_read: 0`, and asserts the file is unchanged. Companion test `g120_l3_append_empty_stdin_without_opt_in_rejects_at_l1` documents that without `--allow-empty-stdin` the L1 guard fires first (exit 65) and the file is preserved.
  3. **G117 edge-case tests** — 3 new tests in `tests/cli_edit.rs`: `edit_unicode_old_new_exact_match` (UTF-8 diacritic `ção` byte-exact, asserts first-occurrence-only contract of single-pair), `edit_crlf_line_endings_preserve_eol_after_replace` (CRLF input + replace + assert byte-for-byte `\r\n` preservation, no normalization collapse), `edit_multi_pair_same_old_appears_twice_applies_both` (multi-pair where both pairs target the same `--old` token — guards against off-by-one where the second pair skips the just-inserted first replacement).
  4. **Regression test update** — `replace_jail_violation_does_not_inflate_counter` in `tests/cli_v012_regressions.rs` was updated to reflect the new G118 contract: exit 126 with `WORKSPACE_JAIL` error envelope and both inside/outside files unchanged. The legacy per-entry behaviour (exit 0 with `files_skipped=1`) is no longer reachable.
- **Consequences**:
  - **+** `replace /etc/passwd` now aborts in microseconds with a clear single error envelope instead of walking the entire filesystem emitting one `JailViolation` per file.
  - **+** G120 L3 is now covered end-to-end. The cross-flag combination (`--append + --expect-checksum + --allow-empty-stdin`) is a documented opt-in for no-op appends on locked files.
  - **+** G117 edge cases are now regression-locked: a future refactor that breaks Unicode exact-match or CRLF preservation or multi-pair sequencing will fail these tests immediately.
  - **+** The replace jail contract now matches `write`/`edit` (fail-fast, exit 126) — one mental model for all mutating commands.
  - **-** The v0.1.12 `replace_jail_violation_does_not_inflate_counter` regression test changed semantics. The test name is kept (the underlying invariant — "jail violation must not inflate the replacement counter" — is preserved) but the assertion body now expects exit 126 instead of `files_skipped=1`.
  - **-** Callers depending on the legacy per-entry tolerance (passing a path that partially escapes, expecting partial processing) must now pre-filter. None observed in the project's own test suite.
- **Alternatives considered**:
  1. Keep per-entry validation and just emit a clearer summary event. Rejected: the walker still wasted I/O and the user-facing diagnostic was still buried. Fail-fast is the correct contract for explicit path arguments.
  2. Add a separate `--allow-jail-violations` opt-in for callers that genuinely want partial processing. Rejected: YAGNI; the legacy behaviour was always a bug, not a feature.
  3. Skip the L3 integration test and rely on unit tests in `cmd_write`. Rejected: G120 was previously delivered without an L3 test precisely because the unit-only coverage was insufficient for a cross-flag combination.
- **Trigger to revisit**: If a future mutating subcommand lands without the resolve-first convention (covered in ADR-0027), promote the textual conformance guard into a shared lint/test harness covering ALL commands. If multi-pair fuzzy cascade grows a 10th strategy, add an edge-case test for it before the strategy is merged.
